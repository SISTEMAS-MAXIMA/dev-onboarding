Paso 03 — Arquitectura

Cómo está construido

El sistema sigue una arquitectura de 3 capas con Blazor WASM como cliente y ASP.NET Core como API.

Capa de Presentación
Blazor WebAssembly
Componentes .razor · IndexedDB · JWT Cookies
Capa de API
ASP.NET Core Web API
Controllers REST · DTOs · Dapper + EF Core · Google OAuth
Capa de Datos
PostgreSQL 17
7 schemas · 60+ tablas · Stored procedures
Infraestructura
Dell T5600 · WSL2 · Docker · Cloudflare Tunnel
Windows Server 2022 · Nginx reverse proxy · SSL automático
Paso 04 — Base de datos

Schemas de PostgreSQL

La base de datos está organizada por dominio de negocio. Cada schema agrupa tablas relacionadas.

product_catalog ~30 tablas · Productos, marcas, atributos, Amazon fees, importación
purchasing 4 tablas · Órdenes de compra, detalle, historial
purchase_tracking 5 tablas · Instancias virtuales, incidencias, tracking
inventory 6 tablas · Instancias físicas, entradas/salidas, auditoría
ecommerce_operations 6 tablas · Business units, canales de venta, modalidades
user_management 3 tablas · Usuarios, roles, asignaciones
app_versions 1 tabla · Control de versiones de la app
Paso 05 — Módulos del ERP

Las pantallas del sistema

El ERP tiene módulos para cada área operativa. Estas son las rutas principales que encontrarás en el código.

Productos

Formulario de 12 pasos para registrar productos con ASIN, precios, atributos, importación, logística.

/form-tab-1 → /form-tab-12

Compras

Registro de órdenes de compra con líneas de producto, tracking, y logística de envío.

/purchasing-details

Recepción

Verificación física en Texas Hub con semáforo de cantidades, incidencias y centro de tareas.

/order-check

Rentabilidad

Calculadora FBA/FBM/ONSITE con referral fees, financiamiento flexible y auditoría paso a paso.

/rentabilidad

Centro de Tareas

Gestión de incidencias y discusiones entre departamentos con prioridades y resolución.

/task-center

Catálogo

Vista paginada de todos los productos con filtros avanzados por checklist de completitud.

/Products-catalog
Paso 06 — Tu primera semana

Checklist de onboarding

Completa estos pasos durante tus primeros días. Haz clic para marcar cada uno.

Clonar el repositorio
git clone desde github.com/SISTEMAS-MAXIMA/MAXIMAERP
Configurar el entorno de desarrollo
Visual Studio 2022+ · .NET 8 SDK · PostgreSQL local o conexión al servidor
Correr el proyecto en localhost
API en https://localhost:7030 · Web en https://localhost:7115
Iniciar sesión con Google OAuth
Tu cuenta de Google debe estar autorizada en el proyecto MAXIMA Wiki
Navegar el formulario de productos
Recorre los 12 pasos (/form-tab-1 al 12) para entender el flujo completo
Revisar la estructura de la base de datos
Conectar a PostgreSQL y explorar los 7 schemas del diccionario
Leer la documentación en Outline
docs.sistemamaxima.com — Infraestructura, Backend, Frontend, Producto
Hacer tu primer commit
Un fix pequeño o mejora de UI — tu bautizo en el codebase
